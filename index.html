Install

Claude Code

One line — runs the bridge straight from GitHub, no clone:

claude mcp add works --scope user -- \
  npx -y github:gentropic/numen --app works --transport fs

Swap --app works for any surface (weir, auditable, …).

Claude Desktop one-click

Build the bundle and double-click it (Settings → Extensions → Install). One multi-surface bridge, run by Desktop's bundled Node — nothing else to install:

npm run mcpb   # → dist/numen.mcpb

Or add an mcpServers entry to claude_desktop_config.json by hand — see the README.

The other half: connect the page

The bridge config is only half the wiring — the surface connects on its side. In Auditable Works: Settings → Agent access to paste a port:token (socket), or Tools → "Connect agent folder…" (fs). The agent then drives the desktop, reaches any package's verbs, and searches the embedded docs — every action gated, consented, and audited.

Full page-side walkthrough + the tool catalog: Auditable Works → Agent Access.

Two transports

The localhost socket (ws/http), and fs — the same protocol over a shared folder, with no port and no extension (the pick for a public-origin PWA, and it reaches another machine if you sync the folder). Design + topology in the SPEC; the fs protocol + its security model in TRANSPORTS.
